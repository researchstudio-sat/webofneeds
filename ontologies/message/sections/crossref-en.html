correspondingRemoteMessageop back to ToC or Object Property ToC

IRI: https://w3id.org/won/message#correspondingRemoteMessage

Points to another message. Messages always exist in two copies: one on the sender's WoN node and one on the receiver's WoN node. The receiver's WoN node copies the message, mints a new URI for it and points to the original message using 'msg:correspondingRemoteMessage'.
has domain
Message c
has range
Message c

hintTargetAtomop back to ToC or Object Property ToC

IRI: https://w3id.org/won/message#hintTargetAtom

The target Atom of the AtomHintMessage. By sending this message, the sender wants to express that the Atom's owner may want to establish a connection with the targetAtom (but first must choose a suitable pair of sockets), or that the targetAtom may just be interesting for the owner for other reasons.

hintTargetSocketop back to ToC or Object Property ToC

IRI: https://w3id.org/won/message#hintTargetSocket

The target Socket of a SocketHintMessage. By sending this message it is suggested to establish a connection between the recipientSocket and the targetSocket.
